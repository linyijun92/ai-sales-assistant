# AI 助销助手 - 产品需求文档（PRD）

## 1. 项目概述

### 1.1 项目背景
- **现有系统**：E 销系统（CRM 系统）
- **痛点**：销售顾问需要手动操作 CRM，效率低下
- **目标**：利用 LLM 能力将 CRM 增强为 AI Agent 应用，提升销售效率

### 1.2 项目愿景
让销售顾问通过语音或文字自然语言，让 AI Agent 智能地操控 CRM 系统，同时以个人助理形式整理客户信息和业务数据。

---

## 2. 核心功能需求

### 2.1 功能 1：自然语言操控 CRM

#### 2.1.1 用户故事
> 作为一名销售顾问，我希望通过语音或文字告诉 AI 我需要做什么操作，让 AI 自动帮我完成 CRM 中的任务，这样我可以专注于与客户的沟通。

#### 2.1.2 功能描述
销售顾问可以通过以下方式操控 CRM：
- **语音输入**：说出需求，如"帮我建立这个客户的档案"
- **文字输入**：输入需求，如"添加一条跟进记录，客户说下周来试驾"

#### 2.1.3 支持的 CRM 操作
| 操作类型 | 说明 | 示例 |
|---------|------|------|
| 客户建档 | 创建新客户档案 | "新建一个客户，张三，138xxxx8888，意向车型 Model Y" |
| 添加跟进记录 | 记录与客户的沟通 | "添加跟进，客户对价格敏感，需要申请优惠" |
| 查询客户信息 | 查找特定客户 | "帮我查一下李四的联系方式" |
| 更新客户状态 | 修改客户跟进阶段 | "将王五的状态改为已预约试驾" |
| 试驾安排 | 安排试驾预约 | "帮客户预约明天上午的试驾" |

#### 2.1.4 验收标准
- ✅ 支持语音输入（语音识别准确率 > 90%）
- ✅ 支持文字输入
- ✅ 能正确识别用户意图（意图识别准确率 > 85%）
- ✅ 能正确调用 CRM API 完成操作
- ✅ 操作结果有明确反馈（成功/失败及原因）

---

### 2.2 功能 2：AI 个人助理 - 数据整理

#### 2.2.1 用户故事
> 作为一名销售顾问，我希望 AI 能帮我整理客户信息和试驾报告，这样我可以快速了解重点客户和跟进优先级。

#### 2.2.2 功能描述
AI Agent 主动整理以下数据：
- **客户信息汇总**：按重要程度、跟进阶段、意向度排序
- **试驾情况报告**：生成试驾汇总，包括试驾人数、车型分布、反馈汇总
- **跟进建议**：基于客户历史数据，提供下一步跟进建议

#### 2.2.3 数据整理场景
| 场景 | AI 处理 | 输出 |
|------|---------|------|
| 每日客户回顾 | 分析当日跟进的客户 | 生成"今日重点客户"列表 |
| 试驾报告整理 | 汇总当日试驾数据 | 生成"试驾日报" |
| 长期未跟进提醒 | 识别超过 7 天未联系的客户 | 生成"待跟进客户"提醒 |
| 意向客户分析 | 分析高意向客户特征 | 生成"意向客户画像" |

#### 2.2.4 验收标准
- ✅ 能自动识别需要整理的数据
- ✅ 生成的报告格式清晰、易读
- ✅ 整理结果符合销售顾问的实际需求
- ✅ 支持导出为常见格式（PDF、Excel）

---

## 3. 非功能需求

### 3.1 性能要求
- **语音识别延迟**：< 2 秒
- **意图识别响应时间**：< 3 秒
- **CRM 操作响应**：< 5 秒
- **并发用户**：支持至少 100 个销售顾问同时使用

### 3.2 可用性要求
- **系统可用性**：> 99.5%
- **平均无故障时间**：> 30 天
- **数据恢复时间**：< 1 小时

### 3.3 安全性要求
- **数据加密**：所有客户数据传输和存储加密
- **访问控制**：基于角色的权限管理
- **审计日志**：记录所有 AI 操作，可追溯
- **隐私保护**：AI 不外泄客户敏感信息

### 3.4 兼容性要求
- **移动端**：支持 iOS 14+ 和 Android 10+
- **CRM 对接**：支持 E 销系统 API v2.0
- **语音识别**：支持中文普通话、方言

## 4. 用户角色

| 角色 | 职责 | 权限 |
|------|------|------|
| 销售顾问 | 使用 AI 操控 CRM，查看整理后的数据 | 读写自己负责的客户数据 |
| 销售主管 | 查看团队数据，导出报告 | 读取团队数据 |
| 系统管理员 | 配置系统，管理用户 | 完全访问权限 |

## 5. 核心工作流程

### 5.1 工作流程 1：语音操控 CRM
```
1. 销售顾问打开应用，点击语音按钮
2. 说出需求（如"新建客户张三"）
3. AI 识别语音 → 转换为文字
4. AI 理解意图（新建客户）
5. AI 提取关键信息（姓名：张三）
6. AI 调用 CRM API 创建客户档案
7. CRM 返回成功/失败结果
8. AI 反馈结果给用户
```

### 5.2 工作流程 2：每日数据整理
```
1. AI 定时扫描 CRM 数据（每日 22:00）
2. 分析当日跟进的客户
3. 分析当日试驾数据
4. 生成整理报告
5. 推送给销售顾问查看
```

## 6. 技术约束

### 6.1 现有系统
- **CRM 系统**：E 销系统
- **API 版本**：v2.0
- **认证方式**：OAuth 2.0
- **数据格式**：JSON

### 6.2 技术栈约束
- **移动端**：Flutter（支持 iOS 和 Android）
- **后端**：Node.js + Express 或 Go
- **数据库**：MySQL（CRM 原有数据库）
- **LLM 集成**：GLM-4.7 或其他商业 API
- **语音识别**：百度语音 API 或阿里云语音 API
